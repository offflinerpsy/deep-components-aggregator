БЫСТРЫЕ SSH КОМАНДЫ ДЛЯ CURSOR IDE

ОСНОВНОЕ ПРАВИЛО: ВСЕГДА добавлять -T после ssh

ПРОВЕРКА СЕРВЕРА:
ssh -T root@89.104.69.77 "curl -s http://127.0.0.1:9201/_version"

ПЕРЕЗАПУСК СЕРВИСА:
ssh -T root@89.104.69.77 "systemctl restart deep-aggregator"

СТАТУС СЕРВИСА:
ssh -T root@89.104.69.77 "systemctl status deep-aggregator --no-pager"

ЛОГИ СЕРВИСА:
ssh -T root@89.104.69.77 "journalctl -u deep-aggregator --lines 10 --no-pager"

ПРОВЕРКА ПОРТОВ:
ssh -T root@89.104.69.77 "lsof -i:9201"

УБИТЬ ПРОЦЕСС ПО ПОРТУ:
ssh -T root@89.104.69.77 "lsof -t -i:9201 | xargs kill -9"

ПРОВЕРКА API:
ssh -T root@89.104.69.77 "curl -s 'http://127.0.0.1:9201/api/search?q=LM317T'"

ЗАГРУЗКА ФАЙЛА:
scp local-file.txt root@89.104.69.77:/remote/path/

ПРОВЕРКА ФАЙЛА:
ssh -T root@89.104.69.77 "grep 'pattern' /path/to/file"

РЕДАКТИРОВАНИЕ ФАЙЛА:
ssh -T root@89.104.69.77 "sed -i 's/old/new/g' /path/to/file"

ПЕРЕЗАГРУЗКА SYSTEMD:
ssh -T root@89.104.69.77 "systemctl daemon-reload"

ПРОВЕРКА ПРОЦЕССОВ:
ssh -T root@89.104.69.77 "ps aux | grep node"

ОСТАНОВКА ВСЕХ NODE ПРОЦЕССОВ:
ssh -T root@89.104.69.77 "pkill -f node"

ПРОВЕРКА ДИСКА:
ssh -T root@89.104.69.77 "df -h"

ПРОВЕРКА ПАМЯТИ:
ssh -T root@89.104.69.77 "free -h"

ПРОВЕРКА СЕТИ:
ssh -T root@89.104.69.77 "netstat -tlnp | grep :9201"
