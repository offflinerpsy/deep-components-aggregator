# UI Smoke Results — Catalog Preview + Assistant

Дата: 2026-01-21

## Что проверяем
- Leaf категория показывает **10** позиций из кэша по умолчанию.
- «Показать ещё» открывает **помощник** (bottom sheet) и не запускает live-search.
- Live-search запускается только когда пользователь вводит `q`.
- При >10 поисков за 60 секунд появляется помощник-антиспам.

## Команды
- Сборка фронта:
  - `cd v0-components-aggregator-page && npm run -s build`
  - Результат: exit code 0 (лог без DYNAMIC_SERVER_USAGE для `/catalog` после `dynamic='force-dynamic'`).

## Ручной чеклист (PASS/FAIL)
1) `/catalog` → выбрать leaf категорию (без `?q=`)
   - Ожидаемо: таблица с результатами (максимум 10) + блок «Показано 10…» + кнопки «Показать ещё» и «Поиск по всем источникам».
2) Нажать «Показать ещё»
   - Ожидаемо: открывается нижний помощник; live-search не стартует.
3) В помощнике нажать «Показать ещё 10 из кэша»
   - Ожидаемо: таблица расширяется, остаёмся в cache-only режиме.
4) Ввести в поиск `LM358`
   - Ожидаемо: стартует SSE `/api/live/search?q=LM358`.
5) Инициировать подряд много поисков (11+ за минуту)
   - Ожидаемо: перед очередным поиском открывается помощник-антиспам с кнопкой «Искать всё равно».
